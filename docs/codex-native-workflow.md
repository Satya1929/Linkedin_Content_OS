# Codex Native Workflow

Codex features are used as accelerators, not as the product backend.

## Development Layer

- Use Codex to implement features, review architecture, run tests, and keep Git history clean.
- Use Browser Use to inspect the local dashboard after frontend changes.
- Use GitHub connector after a remote repository exists.

## Creative Operator Layer

- Create a future `linkedin-content-engine` skill with voice rules, post formulas, scoring rubric, source policy, and lead-magnet rules.
- Use image generation to prototype visual styles and pattern breakers.
- Use Presentations for carousel-style document posts.
- Use Documents/PDF for paid guide production.

## Support Layer

- Use Codex automations for reminders to review drafts, import analytics, and run weekly strategy reviews.
- Keep the product scheduler as the source of truth for actual publishing state.
